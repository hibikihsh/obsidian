現在のGitHubの変更内容を分析して、以下の形式でテスト仕様書を作成してください。
  
## 指示

1. `git diff` コマンドで変更内容を確認する

2. 変更されたファイルとその内容を分析する

3. 各変更について、影響範囲と機能を特定する

4. 以下の形式でテスト項目を作成する

  

## 出力形式

```

テストNo1.

- 項目名/タイトル名: {変更内容を表す項目名}

- 操作手順･条件: {変更箇所を確認するための具体的な操作手順や条件}

- 期待値: {変更後の期待される動作や表示内容}

- 備考: {変更ファイル名、関連する情報など}

  

テストNo2.

- 項目名/タイトル名: {変更内容を表す項目名}

- 操作手順･条件: {変更箇所を確認するための具体的な操作手順や条件}

- 期待値: {変更後の期待される動作や表示内容}

- 備考: {変更ファイル名、関連する情報など}

  

...

```

## テスト項目作成のガイドライン

- フロントエンド変更: UI表示、スタイル、レイアウトの確認項目を作成

- バックエンド変更: API動作、データ処理、ロジックの確認項目を作成

- 設定ファイル変更: 環境動作、ビルド、デプロイの確認項目を作成

- 複数ファイル変更: 統合動作の確認項目を作成

- エッジケース: 境界値、エラーケースの確認項目を含める
